# Hononeko 🐾

**Building the tools I actually use.**

### 💡 Why this exists
Hononeko is my personal sandbox. It’s where I build the small, specific utilities that were missing from my homelab. Most of these projects started because a "standard" solution was too heavy, didn't support a specific feature I needed, or simply because I wanted to see if I could build it better.

### 🛠 The Motivation
* **Solving My Own Problems:** If a port isn't syncing or a notification isn't firing, I’d rather build a sidecar for it than wait for a feature request to be filled elsewhere.
* **Cloud-Native Play:** I run a Kubernetes-heavy environment. Everything here is built to be container-first, lightweight, and easy to manage via GitOps.
* **Learning by Breaking:** Whether it’s experimenting with a new Go framework or refining a Svelte UI, this is where I level up my stack.
